Playwright
You mentioned setting up playwright -- if you're working on web stuff you should set it up right now. It takes less than 5 minutes to get running and it can be extremely helpful. Claude Code has an installable plugin. A few actual use-cases for it:
1- I've built an agentic pipeline that helps me design websites. I have a skill that uses playwright and checks all screen sizes by adjusting the window, taking a screenshot and then inspecting it. Later in the pipeline i use it again to generate thumbnails of designs and build a gallery for me or my client to look at. The gallery large thumbnails of all screen sizes (mobile, tablet, desktop, desktop large) and gives a quick over view
2- Data scraping is a lot easier in a real browser. There's basic stuff like you get JS rendered content but even a playwright driven browser seems to get past a lot of cloudflareish types of blocks
3- Playwright is good for a QA agent. You can have it navigate around and just look for over lapping screens or delays, console errors, etc.

Playwright is super expensive though... it has been great at incinerating my tokens lol!

Claude Code agent teams
I've been trying out Claude's agent teams for the past week or so... It is really interesting. If you haven't tried it i highly recommend checking it out! Basically you can assign a huge chunk of work and a lead agent will build a plan with a prioritized and dependency-ordered task execution requirements. It spawns a team of agents and assigns them. The interesting thing is that the individual team members seem to be just additional claude code processes... there's no shared context and they can communicate with each other through some sort of message passing system! The team lead is not a bottle neck.

If you install tmux and run the latest claude code with:
`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 claude --teammate-mode tmux`

...You can actually watch it spawn new tmux panels and directly interact with the team member agents!

Like playwright, agent teams also burn through tokens but i have found that it gets really huge chunks of work done very quickly. A couple days ago i had it create a birth plan app for new moms in SwiftUI from 1000+ lines long PRD i wrote to see what it could do... 5 hours later it finished and only missed one requirement. The app itself compiled on first run (!), it had a full unit and UI test suite (!!) and all information in the app had medical sources cited (🥼🥼🥼). So cool.

Thanks Radha - glad that you found this useful and shared with your team.

That 'ask and don't assume' is a great way to steer the output - you can add it to your CLAUDE.md file so that the agent will follow that instruction always.

Also, since you like to see the reasoning, you could use Claude Code's "Explanatory" output style which will surface good insights and explains the code as it works. It is easy to configure - just run "/config" command and change the value of "Output style" (use spacebar) and choose "Explanatory".

Happy agentic coding! It is fun! Feel free to reach out if you have any questions as you dive deeper into it!
