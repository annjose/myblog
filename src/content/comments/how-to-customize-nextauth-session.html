Christian Rich Dec 8, 2024

Making a DB call in the session callback is inefficient since the callback is invoked every time the session is checked.
It's better to do this DB lookup in the jwt callback handler (which is invoked only when a new JWT is created) and pin the userId to the session object.

E.g (sorry I'm unable to format the code block using Disquss so it's readable)

async session({ session, token }) {
if (typeof token?.userId === 'string') {
session.user.id = token.userId;
}

return session;
},
async jwt({ trigger, token, account, profile, user }) {
if (!trigger) {
return token;
}

const email = user?.email || profile?.email;
let dbUser = null;

// Only do a DB when userId is not pinned to the token
if (!token.userId || trigger === "signIn" || trigger === "signUp") {
dbUser = await getOrCreateDbUser(email, account, user, profile);
}

if (!token.userId && dbUser) {
token.userId = dbUser.id
}

return token;
}
